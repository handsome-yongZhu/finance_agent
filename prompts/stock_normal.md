# 股票分析报告 - 普通版模板

## 报告结构（6段式）

### 第1段：热点事件切入
以近期的重大事件或股价表现作为切入点，抓住读者注意力。

**要求**：
- 用具体的时间、价格、涨幅数据开篇
- 对比大盘或行业指数，突出个股的超额收益
- 提出核心问题："为什么这只股票能跑赢？"

**示例框架**：
```
2026年X月，{股票名称}（{股票代码}）股价突破¥XXX，创{历史新高/近X年新高}。
回顾近{时间周期}，这只{行业定位}累计涨幅超X%，成为{板块}最亮的星。
{关键事件1}后，股价单周飙涨X%；{关键事件2}落地后，主力资金开启第二轮主升浪。
同一时期，{对比指数}涨幅仅X%——{股票简称}为何能跑出超额收益？
```

---

### 第2段：问题分析（事件背后的机制）
深入分析驱动股价上涨的核心机制，区分不同阶段的驱动力。

**要求**：
- 拆解多波上涨的不同驱动逻辑（预期驱动 vs 基本面驱动）
- 解释资金行为背后的原因（机构建仓、估值切换等）
- 揭示公司发展阶段的跃迁（概念期→放量期→成熟期）

**示例框架**：
```
{X}波上涨背后的驱动逻辑不同。
第一波（{时间}）是{驱动因素}驱动：市场提前交易{预期内容}，叠加{宏观叙事}，{资金类型}快速建仓。
第二波（{时间}）是{驱动因素}驱动：{财报/公告}显示{关键指标}，证明公司从"{阶段A}"进入"{阶段B}"，
估值逻辑从{旧逻辑}切换到{新逻辑}。
两者共同指向——{核心结论}。
```

---

### 第3段：抽象总结（宏观叙事）
将个股行情置于宏观产业趋势中，提升分析的格局和深度。

**要求**：
- 从个股映射到产业范式转移
- 区分周期性机会 vs 结构性重估
- 强调确定性和长期逻辑

**示例框架**：
```
{股票简称}的独立行情映射了整个{产业主题}的范式转移。
{过去时间}之前，{行业}处于"{困境描述}"的尴尬境地。
{转折时间}后，{外部变化}（{具体事件1}、{具体事件2}），
倒逼下游客户——{客户类型}——加速转向{新方案}。

这不是周期性机会，而是结构性重估：
{核心逻辑}从"可选项"变成"必选项"，{股票简称}作为{行业定位}，受益确定性极高。
```

---

### 第4段：结构化拆解（多维分析）
从3个核心维度拆解投资逻辑，提供可验证的分析框架。

**要求**：
- 选择3个最关键的维度（产品/客户/财务/技术/竞争力等）
- 每个维度用数据和事实支撑
- 体现公司从量变到质变的跃迁

**示例框架**：
```
从三个维度拆解{股票简称}的投资逻辑：

❶ {维度1}：{现状描述}，{竞争优势}，{对标对象}，{核心卖点}。

❷ {维度2}：从{旧状态}拓展至{新状态}，{关键指标}占比首超X%，
意味着从"{旧驱动}"转向"{新驱动}"。

❸ {维度3}：{投入指标}长期维持在X%以上，{核心资源}来自{来源}，
{壁垒指标}国内第一，{竞争优势}深厚。
```

---

### 第5段：现状复盘与方法论
评估市场共识，并提示潜在风险，展现客观性。

**要求**：
- 总结市场当前的主流观点
- 警惕3个具体的风险点
- 风险要具体且可量化

**示例框架**：
```
市场当前共识是"{市场观点}"，支撑逻辑是"{核心逻辑}"。
但需警惕三个潜在风险：

❶ {风险1}：当前{估值指标} X倍，已反映{预期内容}，若{触发条件}，{后果}。

❷ {风险2}：{竞争对手/技术变化}，{公司}的{竞争优势}可能{负面影响}。

❸ {风险3}：{流动性/政策/供应链}风险，{具体威胁}，一旦{触发条件}，{影响}。
```

---

### 第6段：未来观察点与操作建议
提供可跟踪的观察指标和具体的操作策略。

**要求**：
- 列出3个关键观察指标（带时间节点）
- 给出明确的技术位和操作建议
- 强调风控纪律

**示例框架**：
```
关键观察：
❶ {时间节点}的{事件/数据}（关注{具体指标}）；
❷ {业务进展}的{验证点}（{订单/中标/合作}能见度）；
❸ {技术/产能}进展（{具体突破点}）。

操作上，¥{价格1}为短期多空分水岭，站稳可看¥{价格2}（{目标位说明}）；
失守¥{价格3}则回吐涨幅，考验{支撑位说明}。

建议{操作策略}，单次不超X%仓位，严守止损。
```

---

## 输出要求

1. **字数控制**：600-800字
2. **数据准确**：所有数据必须有来源，标注[来源]
3. **客观中立**：不做"买入/卖出"等投资建议，只做分析
4. **可读性强**：使用emoji、分段、加粗等提升阅读体验
5. **格式规范**：使用Markdown格式

---

## 图表要求（⚠️ 必需）

虽然是普通版，但**必须包含K线图**：
- ✅ **必须生成K线图**并保存为：`images/{股票代码}/kline.png`
- ✅ **必须在报告中引用**：`![K线图](/images/{股票代码}/kline.png)`
- ✅ 放在"技术面速览"或"走势图"部分
- ❌ 禁止使用base64图片
- ⚠️ 没有图表的报告不完整

## 禁止事项

❌ 禁止使用"强烈推荐买入"等诱导性语言
❌ 禁止编造数据或消息
❌ 禁止抄袭其他分析师报告原文
❌ 禁止忽视重大风险因素
❌ **禁止不包含图表**（每份报告必须有K线图）
